# ぬの しわけ サポート

布の色が見分けにくい利用者向けに、写真からしわけ先を大きな文字と音声で案内する Web アプリです。保存はすべて端末内の `IndexedDB` に行い、サーバーやクラウド同期は使いません。

## 特徴

- `とる -> たしかめる -> この端末に保存` の 3 ステップで使えます。
- 色分析はブラウザ内で完結し、ネットワークやログインに依存しません。
- 履歴は同じ端末・同じブラウザ内にだけ保存されます。
- 職員向けに `fabric-history.v2.json` のエクスポート / インポートを用意しています。
- ルーティングは `HashRouter` を使うため、`/#/history` を含む静的配信だけで動作します。
- 配布時は `vercel.json` の CSP とフレーム拒否ヘッダーで静的サイトを保護します。

## 技術スタック

- React 19 + TypeScript
- Vite
- Material UI
- `ml-kmeans`
- `IndexedDB` + `idb`

## ローカル実行

```bash
npm install
npm run dev
```

ブラウザで表示された URL を開くと利用できます。環境変数は不要です。

## ビルド

```bash
npm run build
```

生成された `dist/` を静的ホスティングへ配置すれば動作します。

## 配布

- 既定の配布先は `Vercel` の静的ホスティングです。
- `vercel.json` に CSP、`X-Frame-Options: DENY`、`Permissions-Policy` などのセキュリティヘッダーを定義しています。
- `HashRouter` を使っているため、GitHub Pages や単純な静的サーバーでもそのまま配布できます。

## 運用メモ

- このリポジトリはブラウザ内完結の静的アプリとして運用します。
- 旧 Firebase / Functions 構成は廃止対象です。別環境にデプロイ済みの Functions や Cloud Storage 書き込み経路が残っている場合は、リポジトリ整理と合わせて停止または削除してください。
- 新しいバックエンド依存は追加しない前提です。履歴保存は `IndexedDB` のみを使います。

## データ保存

- 保存内容: 解析結果 + 640px JPEG プレビュー
- 保存上限: 100 件
- 超過時: 古い履歴から自動削除
- 共有範囲: 端末・ブラウザ単位

## テスト

```bash
npm test -- --run
```

`fake-indexeddb` を使ってローカル履歴の保存・削除・取り込みを検証します。
